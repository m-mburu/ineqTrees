# Publishing R package sites on GitHub Pages with pkgdown

This note records the process used to publish the `ineqTrees` package site at:

```text
https://m-mburu.github.io/ineqTrees/
```

Use the same pattern for future R package repositories.

## Goal

For each R package repo, create a pkgdown website and publish it as a project
site under the main GitHub Pages account site:

```text
https://m-mburu.github.io/<package-name>/
```

For example:

```text
https://m-mburu.github.io/ineqTrees/
https://m-mburu.github.io/anotherPackage/
```

The main personal site at `https://m-mburu.github.io/` can then link to each
package site from a "Packages", "Software", or "Projects" section.

## 1. Work on an experiment branch

Create a branch before changing the package website setup:

```bash
git switch -c pkgdown-site
```

This keeps the pkgdown workflow isolated until the GitHub Actions build works.

## 2. Add pkgdown configuration

Create `_pkgdown.yml` in the repo root.

Example:

```yaml
url: https://m-mburu.github.io/ineqTrees/

template:
  bootstrap: 5

development:
  mode: release

home:
  title: ineqTrees
  description: Tree-based decomposition of health inequality.

navbar:
  structure:
    left: [intro, reference, articles]
    right: [search, github]
```

For another package, change:

- `url`
- `home.title`
- `home.description`
- any `articles` sections if you want to organize vignettes manually

## 3. Update DESCRIPTION

Add the pkgdown URL before the GitHub repo URL:

```r
URL: https://m-mburu.github.io/ineqTrees/, https://github.com/m-mburu/ineqTrees
BugReports: https://github.com/m-mburu/ineqTrees/issues
```

For another package, replace `ineqTrees` with the package/repo name.

## 4. Ignore generated site files

The `docs/` directory is generated by pkgdown. It does not need to be committed
when deployment is handled by GitHub Actions.

Add to `.gitignore`:

```text
/docs/
```

Add to `.Rbuildignore`:

```text
^_pkgdown\.yml$
^docs$
```

## 5. Add the GitHub Actions workflow

Create `.github/workflows/pkgdown.yaml`.

This is the working pattern used for `ineqTrees`:

```yaml
on:
  push:
    branches: [main, master]
  pull_request:
  workflow_dispatch:

name: pkgdown.yaml

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest

    env:
      GITHUB_PAT: ${{ secrets.GITHUB_TOKEN }}

    steps:
      - uses: actions/checkout@v4

      - uses: r-lib/actions/setup-pandoc@v2

      - uses: r-lib/actions/setup-r@v2
        with:
          use-public-rspm: true

      - uses: r-lib/actions/setup-r-dependencies@v2
        with:
          extra-packages: |
            any::pkgdown
            local::.
          needs: website

      - name: Build site
        run: pkgdown::build_site_github_pages(new_process = FALSE)
        shell: Rscript {0}

      - name: Upload artifact
        if: github.event_name != 'pull_request'
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs

  deploy:
    if: github.event_name != 'pull_request'
    needs: build
    runs-on: ubuntu-latest

    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}

    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

Important detail:

```yaml
extra-packages: |
  any::pkgdown
  local::.
```

The `local::.` line installs the current package on the GitHub Actions runner.
Without this, pkgdown may fail while building reference pages with:

```text
there is no package called '<package-name>'
```

This is equivalent in spirit to running `devtools::install()` before building
the website, but it uses the dependency installation system from `r-lib/actions`.

## 6. Optional: test on an experiment branch

To test before merging, temporarily include the experiment branch in the
workflow trigger:

```yaml
on:
  push:
    branches: [main, master, pkgdown-site]
```

To avoid deploying from the experiment branch, gate artifact upload and deploy:

```yaml
if: github.event_name != 'pull_request' && (github.ref_name == 'main' || github.ref_name == 'master')
```

Use that condition on both the `Upload artifact` step and the `deploy` job.

Once the workflow is proven, remove the experiment branch from the trigger or
leave it only if you intentionally want future branch tests.

## 7. Build locally before pushing

From the package root, run:

```r
pkgdown::build_site(new_process = FALSE, install = FALSE)
```

If the package is not already loadable locally, install or load it first:

```r
devtools::install()
```

or:

```r
pkgload::load_all(export_all = FALSE)
```

The local build writes to `docs/`. That directory is ignored, so it is only a
preview.

## 8. Fix pkgdown warnings before publishing

During the `ineqTrees` setup, pkgdown built successfully locally but warned
about missing alt text for generated figures. These warnings were fixed by
adding `fig.alt` to the relevant R Markdown chunks.

Example:

````markdown
```{r readme-tree-plot, fig.width=15, fig.height=9, fig.alt="A concentration-index tree fitted to under-five mortality in the Kenya example data."}
readme_tree_plot(fit_tree, kenya, "deadu5_num")
```
````

For generated README files, update both:

- `README.Rmd`
- `README.md`, or rerender `README.Rmd` to regenerate `README.md`

If a vignette is generated by `fusen`, update both the generated vignette and
the source file in `dev/` if needed.

## 9. Push and inspect GitHub Actions

Commit and push the branch:

```bash
git add _pkgdown.yml .github/workflows/pkgdown.yaml DESCRIPTION .gitignore .Rbuildignore
git commit -m "Add pkgdown site workflow"
git push -u origin pkgdown-site
```

Check runs with GitHub CLI:

```bash
gh run list --branch pkgdown-site --limit 5
gh run watch <run-id> --exit-status
gh run view <run-id> --log-failed
```

Common failure:

```text
there is no package called '<package-name>'
```

Fix:

```yaml
extra-packages: |
  any::pkgdown
  local::.
```

## 10. Enable GitHub Pages for the repo

The deployment can fail even after the site builds if GitHub Pages has not been
enabled for the repository.

Failure message:

```text
Creating Pages deployment failed
HttpError: Not Found
Ensure GitHub Pages has been enabled
```

Fix in the browser:

1. Open the package repository on GitHub.
2. Go to `Settings` > `Pages`.
3. Set the source/build option to `GitHub Actions`.
4. Save the setting.

Fix with GitHub CLI:

```bash
gh api --method POST repos/m-mburu/ineqTrees/pages -f build_type=workflow
```

For another repo, replace `ineqTrees`:

```bash
gh api --method POST repos/m-mburu/<repo-name>/pages -f build_type=workflow
```

You can confirm the Pages configuration with:

```bash
gh api repos/m-mburu/<repo-name>/pages --jq "{html_url, status, build_type, https_enforced}"
```

Expected `build_type`:

```text
workflow
```

## 11. Merge and rerun if needed

After the branch workflow succeeds, merge the branch into `main`.

If deployment failed before Pages was enabled, rerun the failed workflow:

```bash
gh run rerun <run-id>
gh run watch <run-id> --exit-status
```

For `ineqTrees`, after enabling Pages and rerunning the workflow, both jobs
passed:

```text
build  passed
deploy passed
```

The final site URL was:

```text
https://m-mburu.github.io/ineqTrees/
```

## 12. Repeat for future packages

For each future package:

1. Make sure it is a valid R package with `DESCRIPTION`, `NAMESPACE`, `R/`,
   `man/`, and any vignettes.
2. Add `_pkgdown.yml` with the correct package URL.
3. Add or update `DESCRIPTION` with the package website URL.
4. Add `docs/` and `_pkgdown.yml` to ignore/build-ignore files.
5. Add `.github/workflows/pkgdown.yaml`.
6. Make sure the workflow installs the local package with `local::.`.
7. Push an experiment branch and watch the Actions run.
8. Enable Pages with `build_type=workflow`.
9. Merge to `main`.
10. Confirm the site works at `https://m-mburu.github.io/<package-name>/`.

## Useful commands

Local build:

```bash
Rscript -e "pkgdown::build_site(new_process = FALSE, install = FALSE)"
```

List workflow runs:

```bash
gh run list --workflow pkgdown.yaml --limit 5
```

Watch a workflow:

```bash
gh run watch <run-id> --exit-status
```

Show failed logs:

```bash
gh run view <run-id> --log-failed
```

Enable GitHub Pages for Actions:

```bash
gh api --method POST repos/m-mburu/<repo-name>/pages -f build_type=workflow
```

Check GitHub Pages:

```bash
gh api repos/m-mburu/<repo-name>/pages --jq "{html_url, status, build_type, https_enforced}"
```
